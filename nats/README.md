## Ключевые моменты:

### 1. Только Consumer работает с механизмами повторов и DLQ

// Consumer решает:
$consumer->ack($msg);  // ✅ Успех - удалить из JetStream
$consumer->nak($msg);  // 🔄 Ошибка - оставить в JetStream для повтора
$consumer->term($msg); // 🚫 Критическая ошибка - не повторять
2. Stream (хранилище) просто хранит сообщения

Роль	Описание
Publisher (Отправитель)	Просто кидает сообщения, ничего не знает о JetStream/DLQ
Администратор	Настраивает стримы и консьюмеров
Consumer (Получатель)	ЕДИНСТВЕННЫЙ, кто:
• Работает с механизмом подтверждений (ack/nak)
• Принимает решение о повторных попытках
• Отправляет сообщения в DLQ
• Удаляет сообщения из основного стрима после успеха или отправки в DLQ
